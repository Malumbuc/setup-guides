How I Set Up My Claude Code Presets

A guide to replicating my Claude Code configuration for persistent, project-aware AI assistance.

↗ View on GitHub Gist

What Are Presets?

Claude Code supports two main types of persistent configuration:

  • CLAUDE.md — Instructions you write that Claude loads every session
  • Auto Memory — Notes Claude writes itself as it learns your project

1. CLAUDE.md (Your Instructions to Claude)

This is the most important file. Create it in your project root and Claude will load it automatically at the start of every session.

Scopes

FileAffects
/your-project/CLAUDE.mdThat project (shareable via git)
~/.claude/CLAUDE.mdAll your projects (personal, not shared)

What to Put In It

Good CLAUDE.md files answer:

  • What is this project? (overview, stack, architecture)
  • What files exist and what do they do?
  • What patterns does the code follow?
  • What are the git/workflow rules?

My Project's CLAUDE.md

# CLAUDE.md

## Project Overview
Vanilla HTML/CSS/JavaScript project — no build system, no dependencies.
Open any .html file directly in a browser.

## Files
- `index.html` — Two-player Tic Tac Toe with score tracking and win animations
- `tic-tac-toe.html` — Simpler two-player variant
- `tic-tac-toe-ai.html` — Single-player vs AI (Easy/Hard difficulty)

## Git Workflow
Commit work regularly. When pushing, create a new public repo under the
Malumbuc GitHub account (named after the file/feature), not the existing remote.

## Architecture
All files are self-contained single-page apps (HTML + CSS + JS in one file).

Shared patterns:
- Board state: flat 9-element array (null/'' for empty, 'X'/'O' for played)
- Win detection: hardcoded WINS array of 8 winning index triplets
- DOM cells: data-index or data-i attributes map to board array
- CSS classes toggled: taken, x, o, winner/win

Tips for Writing CLAUDE.md

  • Keep it under 200 lines — Claude loads the whole file every session
  • Be specific — "Use 2-space indentation" beats "format code nicely"
  • Commit it — checking CLAUDE.md into git shares it with your whole team

2. Auto Memory (Claude's Own Notes)

Claude automatically takes notes about your project and saves them across sessions. You don't write these — Claude does.

Where It Lives

~/.claude/projects/<project-id>/memory/MEMORY.md

What Gets Saved

  • Key architectural decisions and patterns it discovers
  • Solutions to recurring problems
  • Your workflow preferences it picks up on
  • Important file paths and project structure

How to Control It

To ask Claude to remember something explicitly, just say:

"Remember to always create a new repo under Malumbuc when pushing."

To ask Claude to forget something:

"Stop remembering that, remove it from memory."

To view everything Claude has loaded, run /memory in any session.

Enable / Disable Auto Memory

In .claude/settings.json in your project:

{
  "autoMemoryEnabled": true
}

3. Quick Setup Steps

  1. Create CLAUDE.md in your project root with your project overview, file descriptions, architecture patterns, and workflow rules.
  2. Commit it — git add CLAUDE.md && git commit -m "Add Claude Code project instructions"
  3. Start a Claude Code session — it will auto-load your CLAUDE.md and begin building memory as you work.
  4. Optional: Create ~/.claude/CLAUDE.md for personal preferences that apply across all your projects.

Resources

  • Claude Code docs — full documentation
  • Run /memory in any session to inspect what Claude has loaded
  • Run /help for a full list of Claude Code commands

How I Connected Claude Code to GitHub

Claude Code uses the GitHub CLI (gh), which you install and authenticate once. After that, Claude can create repos, push code, create Gists, open PRs, and more.

↗ View on GitHub Gist

Step 1: Install the GitHub CLI

On macOS with Homebrew:

brew install gh

Or download from cli.github.com.

Step 2: Authenticate with GitHub

Run:

gh auth login

Follow the prompts:

  • Select GitHub.com
  • Select HTTPS as your preferred protocol
  • Select Login with a web browser (easiest)
  • Copy the one-time code shown, press Enter, and paste it in the browser

gh stores your credentials in your system keyring — no token files to manage.

Step 3: Verify It Worked

gh auth status

You should see something like:

github.com
  ✓ Logged in to github.com account YourUsername (keyring)
  - Token scopes: 'gist', 'read:org', 'repo'
Make sure gist and repo are in your token scopes. If they're missing, re-run gh auth login and select those scopes.

What Claude Can Do After This

TaskCommand Claude uses
Create a new repogh repo create
Push code to GitHubgit push (via gh-configured auth)
Create a Gist (shareable link)gh gist create
Open a pull requestgh pr create
View issuesgh issue list

My Setup

  • Account: Malumbuc
  • Protocol: HTTPS
  • Auth method: Web browser (one-time code)
  • Credentials stored: macOS keyring (automatic, no token management needed)
  • Scopes granted: gist, read:org, repo

Notes

  • You only need to do this once per machine — credentials persist
  • Claude never sees your token directly; it just runs gh commands
  • To log out: gh auth logout
  • To switch accounts: gh auth login again with a different account

How I Set Up Cursor

Cursor is an AI-powered code editor built on VS Code. This guide covers installation, GitHub connection, and configuring Claude as the AI model.

↗ View on GitHub Gist

Step 1: Install Cursor

Install via Homebrew:

brew install --cask cursor

Launch from Spotlight (Cmd + Space → type "Cursor") or from /Applications/Cursor.app.

Alternative: Download the installer directly from cursor.com if you prefer not to use Homebrew.

Step 2: Connect to GitHub

Cursor is built on VS Code, so GitHub integration works the same way.

  1. Open Cursor
  2. Click the Accounts icon in the bottom-left sidebar (person icon)
  3. Click Sign in with GitHub
  4. A browser window opens — authorize Cursor in GitHub
  5. You're redirected back and logged in

Once connected you can clone repos (Cmd + Shift + P → "Git: Clone") and push/pull from the Source Control panel.

Step 3: Set Up Claude as the AI Model

By default Cursor may use OpenAI models. To switch to Claude:

  1. Open Cursor Settings: Cmd + , or go to Cursor menu → Settings → Cursor Settings
  2. Go to the Models tab
  3. Under AI Models, enable the Claude models you want (e.g. claude-sonnet-4-5, claude-opus-4-5)
  4. Set a default model — I use Claude Sonnet for everyday tasks

Using Your Own Anthropic API Key (Optional)

  1. Go to Cursor Settings → Models
  2. Find the Anthropic API key field
  3. Paste your API key from console.anthropic.com

Step 4: Key Features to Know
